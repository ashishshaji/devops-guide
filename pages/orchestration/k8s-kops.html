Production Management

Kubernetes Operations (KOPS)

While Minikube is for local development, KOPS is the enterprise standard for managing production-ready clusters on AWS. It automates provisioning, scaling, and updates.

State Storage

Uses S3 buckets as the cluster's source of truth.

IAM Control

Granular AWS permissions for node security.

provisioning.sh 
# Export state store location
export KOPS_STATE_STORE=s3://my-k8s-bucket

# Create cluster configuration
kops create cluster --name=pro-cluster.k8s.local \
  --zones=us-east-1a --node-count=3 --node-size=t3.medium

# Apply and build infrastructure
kops update cluster pro-cluster.k8s.local --yes

The Smallest Unit: Pods

The fundamental building block of Kubernetes execution.

Isolation

Each Pod gets its own IP address and network namespace, isolating it from others.

Shared Resources

Multiple containers in a Pod share the same storage volumes and localhost.

Ephemeral

Pods are not permanent. If they fail, Kubernetes replaces them with new ones.

Essential Developer Cheatsheet

Inspection

  • kubectl get pods
  • kubectl describe pod [name]

Deployment

  • kubectl apply -f app.yaml
  • kubectl delete -f app.yaml

Debugging

  • kubectl logs [pod-name]
  • kubectl exec -it [pod] -- sh
